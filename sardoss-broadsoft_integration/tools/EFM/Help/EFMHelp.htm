• TAR Pre-configuration

    As a first step in activating EFM service, you should configure the TAR subinterface so that no one else can take it during the activation. Do a last minute check in the TAR to make sure the interface is clear before you configure anything.

    1. Log into the TAR and enter 'enable' mode.
    2. Enter "show run | begin Port-channel1".
    3. Scroll through the list of EFM (Port-channel1) subinterfaces and make sure your interface has not been configured already.
    4. If interface is clear, configure interface using script provided.
    5. Double-check the interface after configuration to make sure everything looks OK and nothing failed.

  • Verify IAD Connectivity

    Prior to changing the config on either TAR or the IAD, a number of checks should be done to verify that customers’ IAD is functioning correctly. This will ensure that any configuration change could be the only reason a customers IAD is not performing as expected.

    1. Execute "show mgcp" in the IAD.
    2. Make sure the MGCP Admin State and Oper State are "ACTIVE".
    3. Trace to the MGCP call-agent.

    4. Trace to the internet.

  • TAR Serial IP Switchover

    To make the customer's transition to EFM as smooth and quick as possible, use the script provided to quickly change the IP on the current TAR interface to the dummy IP and make the old IP the secondary IP on the interface. If there are no active calls, this should not cause a noticeable service interruption and should allow you to verify you have a good connection to the IAD on the new IP. When that's established, you'll have a backup connection into the IAD so you can configure the IAD subinterfaces without losing connection.

    IMPORTANT NOTE:
    Make sure you highlight the entire config when copying to your clipboard. If the script is copied line-by-line, it will cause a service interruption and you may lose connection to the IAD.








































  • IAD Serial IP Switchover

    This is the same as the TAR switchover, just in the IAD this time. One additional change removes restrictions on incoming virtual terminal connections to allow telnet access from the TAR.

    IMPORTANT NOTE:
    Make sure you highlight the entire config when copying to your clipboard. If the script is copied line-by-line, it will cause a service interruption and you may lose connection to the IAD.








































  • Telnet to IAD from TAR

    Telnet directly to the IAD from the TAR. Use this connection to complete the rest of the IAD configuration.

    IMPORTANT NOTE:
    Be VERY careful and know which device you are working in at all times. Telnetting to the IAD from the TAR can create some confusion. ALWAYS look at the hostname of the device before you configure anything or execute a command.

  • Connect EFM Gear to IAD

    This step will take down the customer's voice and data connections until configuration is complete. Have the on-site tech verify the customer is ready to go down and have the tech hook up the CPE gear to the IAD and to the customer's network.

    IMPORTANT NOTE:
    Make sure the tech does not take down the T1 connection to the IAD at this point. You need this connection to complete the configuration.

      Procedure for connecting EFM box to IAD
    1. Disconnect customer's network from FastEthernet0/0 port on IAD.
    2. Connect customer's network to Ethernet2 port on EFM box.
    3. Connect Ethernet1 port on EFM box to FastEthernet0/0 on IAD (crossover cable).








































    •

  • Configure IAD Subinterfaces

    Use the script to configure the IAD subinterfaces. The script defines the interface for the upstream connection to the Cbeyond network. It moves the serial IP from the serial (or multilink) interface to the new subinterface. It also defines the interface for the customer's network and repoints the NAT for the customer's private IPs.








































  • Move TAR Serial IP

    Use the script in the TAR to move the customer's serial IP from the old interface to the new EFM subinterface and to repoint the IP route for the customer's public IP.








































  • Change IAD Default Route

    Use the script in the IAD to repoint the default route to the EFM connection. Notice this IP route also contains a next-hop address. This is necessary for a FastEthernet connection and CANNOT be omitted from the config.








































  • Verify Services

    All services should be back up and running now. Verify connectivity to IAD and check connections to the call agent and internet. Have the on-site technician check all of the phone lines and run a speed test to verify new upload & download times.

    1. Trace to the internet from the IAD to confirm internet connectivity.

    2. Have the tech use 'speakeasy.net' or 'speedtest.net' to test the data connection speed. This can usually be done from one of the customer's PCs, but if the speed is slower than expected, the speed will need to be tested with the tech's laptop plugged directly into the Ethernet2 port on the EFM box.

        Expected data speeds
      • BVI = between 1.5 and 2.0 MB/sec
      • BVII = between 3.0 and 4.0 MB/sec
      • BVI = between 4.5 and 6.0 MB/sec

    3. Status terminations or trunks in the call agent to verify voice services.

    4. Make test calls to and from the customer's phones.
